植物记（一）

前言

上周末去了植物园，原本是想去赏腊梅的，走了很久的路看到的腊梅略感失望。比起16年年初在西安小雁塔看到的那几株腊梅花，品相差太多，香气也不足，傲霜风骨全然不见。冬天植物园很是萧索，能看到的绿植也不多，只粗粗拍了些照片，方便回来查阅。

Ps：植物园里的鲤鱼真的很大个，第一次见到这么大的鲤鱼，吓到了。

植物列表

虎耳草

虎耳草科植物。喜阴凉潮湿，土壤要求肥沃、湿润，以密茂多湿的林下和阴凉潮湿的 坎壁上较好。

全草入药；微苦、辛，寒，有小毒，凉血解毒。

祛风，清热，凉血解毒。治风疹；湿疹，中耳炎，丹毒，咳嗽吐血，肺痈，崩漏，痔疾。

菊三七

为菊科土三七属植物。 以根或全草入药。秋冬挖根，除去残茎、须根及泥土晒干。夏秋采全草，洗净，鲜用或晒干。

甘、微苦，温。

止血，散瘀，消肿止痛，清热解毒。

适用于吐血，衄血，咯血，便血，崩漏，外伤出血，痛经，产后瘀滞腹痛，跌打损伤，风湿痛，疮痈疽疔，虫蛇咬伤。

2016-Review

前言

昨天新赛季更新在公司加班，实现了工作跨年。1点多回的家，一直睡到11点起来做饭，吃完饭逛了城西银泰又去了沃尔玛，晚上跑步8公里，帮妈妈处理了照片在淘宝下了单。一直一直在处理着琐碎的事情，本该昨晚完成的年度总结，只能今晚写了，现在已是2017年1月1日23时30分，总结能不能在今天完成仍是个未知数。这会儿广播里播放的朴树的白桦林，很放松的氛围。

工作

16年的中心仍然是工作，摩羯大约真的是工作狂。

6月初学习了react和redux，然后花了近一个的时间，重构了对战平台。后期就是各种功能的新增，各种加班，9月之后工作就恢复正常状态，少加班，主要工作是维护网站和平台。

回顾这一整年，唯一能让我稍有自豪感的只有react重构平台，其他总是不能令自己满意。

这一年成长得比较曲折，如果给自己打分，工作方面打80的话，但与工作相关方面就比较糟糕了。

脾气的燃点越来越低，程序大约前世欠了产品很多钱吧……纵然产品不让人满意，我有时的引导也做得不好，这点在17年需要改进。

另外一点，我一直觉得自己非常好沟通，今年也被磨了尖锐得有点嫌弃自己了，某些时候，会放弃陈述沟通，产品说什么就是什么。今年，我要坚持自己的是非观，不合理就是不合理，不再给自己挖坑了。

关于工作，就写这些了，历练加成长是永恒的主题。

学习

阅读

今年书读得真的很少，翻了翻豆瓣，少的可怜，不论是技术书籍还是闲书。

好的书籍使人明理，身心愉悦，身心成长。所以好的阅读习惯还是要坚持的。

理财

从来没有理财观念，甚至消费观都一塌糊涂。

从前，穷了好多年，一块钱想当两块钱花。现在独立自主，经济好了，花钱就大手大脚，仿佛要把从前错过的都买回来。但其实，很多东西都不是自己想要的了。所以，今年浪费了不少钱财，家里堆积了不少无用之物。好在下半年，觉悟了，决定这一生都遵循less is more的准则生活下去了。弯路走了不少，好在还能回头。

下半年关注了越女，跟着她学了不少理财知识。现在慢慢在研究基金，分散资金，而不是像从前那样鸡蛋放一个篮子里面了。

穿搭

哈哈，回顾我之前的穿衣风格，简直是一场灾难片。

今年关注了好几个穿搭的公众号，进步斐然。在黑白灰三色向顺色搭配的过渡中。买衣服也比较慎重了，基本是百搭单品。这是16年比较值得欣慰的事情之一了。

生活

运动

这一年，我深切贯彻了“三天打鱼，两天晒网”的理念。

6-8月，借口是加班好累啊。

9月比较好，一整个月都有坚持运动。所以9月是我一年中最瘦的时候，53kg。

10-12月，废人一个。

不多说，keep keep keep！

减肥

状态同上。惨败。

旅行

16年，独自去了一个长线，和艳姐结伴游了几座山，完成了大朝台。

1月，独自去了西安和洛阳，一直以来的古都情节得到了回应。西安这座城市，虽然地处西北，却给了我类似杭州的温暖的感觉，我挺喜欢。秦岭太白将是17年要去的地方。洛阳的节奏就慢多了，匆匆一日，玩得并不尽兴。来年春天，可以去洛阳小住。华山是此行的最后一站，很喜欢。清晨6:30从玉泉院上山，傍晚18:30智取华山道下山，全程12个小时。运气特别好，遇到了同行的驴友，于是结伴游玩了华山。下山时没有尽头的绝望也在相互鼓励中消弭了。看到山下一排排的灯迹时的喜悦之情我一生难忘。

4月，跟艳姐去爬了泰山，后山的风景较赞。

7月，跟艳姐去了武夷山。

8月，跟艳姐去了雁荡山，略失望。从雁荡山开始，我决定以后都不走旅游线路，改走户外线了。

9月，中秋独自去大朝台。五台山，会是我每年都去一次的地方。运气很好，朝台两天的天气都特别的赞，途中还遇到了有趣的驴友，西台和驴友结伴走完下面的行程。五台山忘川谣的客栈小哥也很赞，是个比较有情怀的人。9月不是朝台的最佳时机，但是依然看到大片的草甸，牛啊，马啊，还有天空大朵大朵的云。前段路都是我一人在行走，天地间只有一人的感觉，很特别。我喜欢五台山，有生之年还是要去的。

我特别喜欢一个人的行程，想去一个地方，做好攻略，了解文化和民俗，在我看来是特别有趣又美好的事情，我享受整个过程。今年在五台山，我却希望旁边有一个伴侣，我们能共享美景，共同承载回忆。找个伴侣吧，我对自己说，放弃你那固执的独身主义吧。这算成长么？我也不知道。

后记

这篇总结写得真心不容易，昨晚上写了一个小时的总结却因为没有及时保存，直接没了。后来有写了一篇，因为程序的问题加上没有备份，又没了。。。。。。大写的心疼我自己。今天早起在编辑器里面从新写了一份，这次没有侥幸，一定不会丢失了。

背景 - 为回调函数而生

jQuery 1.5.0版本之前，回调函数的功能非常弱，deferred对象就是jQuery的回调函数解决方案。

使用场景

1.ajax链式用法

3.多个操作指定同一回调

通常，项目中会有如下需求，当两个或两个操作都执行成功时，才执行一个函数，这时使用deferred对象就可以了。

4.普通操作的回调函数接口

deferred对象应用广泛，这一套回调函数接口既可用于ajax操作又可用于本地操作，异步同步皆可。

deferred对象有三种执行状态--未完成、已完成、已失败

已完成（resolved），deferred对象立即调用done()方法指定的函数 已失败，调用fail()方法指定的函数 未完成，则继续等待，或调用progress()方法指定的函数

deferred.reject()方法用于将执行状态从未完成改为已失败，触发fail()方法
deferred.promise()方法，在原来的deferred对象上返回另一个deferred对象，这个对象只开放与改变执行状态无关的方法(如done()和fail())，而不开放与改变执行状态有关的方法（如resolve()方法和reject()方法），从而使执行状态不可更改

Javascript 获取服务器时间

背景

单页操作中，通常我们需要获取当前的时间节点，如倒计时操作。Javascript提供了Date()对象来获取客户端时间，但是客户端时间并不是恒定的，用户可以自行更改时间，这就导致每个用户的时间不一致。此时，我们需要一个恒定的服务器时间来保持用户见到的时间一致。
服务器的时间，通常由后台程序通过php、jsp等返回，但仅仅为了一个时间架一个后台程序，这是浪费生命的操作，绝不提倡。
目前，发现的最好的方法是ajax获取http的头部信息，由此来获取服务器时间。

原理

下图为服务器响应请求返回的静态页面的HTTP头部信息，可以直接用ajax获取头部信息里的Date信息。

实现

Javascript版
var xhr = new XMLHttpRequest();   
if( !xhr ){   
  xhr = new ActiveXObject('Microsoft.XMLHTTP');
}   
xhr.open('HEAD',location.href,true);   
xhr.onreadystatechange=function(){   
    if( xhr.readyState == 4 && xhr.status == 200 ){   
        alert(xhr.getResponseHeader("Date"));   
    }   
}   
xhr.send(null);
